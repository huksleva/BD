### Проектирование БД с учетом 3НФ

На основе исходного набора данных о продаже спортивных товаров была спроектирована реляционная база данных, соответствующая требованиям третьей нормальной формы (3НФ).

---

# Исходные данные

В таблице присутствовали следующие данные:

- товары;
    
- заказы;
    
- заказчики;
    
- количество товаров в заказе;
    
- адрес и телефон заказчика.
    

---

# Нормализация данных

Для устранения избыточности данные были разделены на несколько связанных таблиц.

---

# Таблица «Customer»

|Поле|Описание|
|---|---|
|CustomerNum|Номер заказчика|
|CustomerName|ФИО заказчика|
|CustomerAddress|Адрес|
|CustomerPhone|Телефон|

### Первичный ключ

- CustomerNum
    

### NOT NULL

- CustomerName
    
- CustomerAddress
    
- CustomerPhone
    

---

# Таблица «Product»

|Поле|Описание|
|---|---|
|CatalogNum|Каталожный номер|
|Product|Название товара|
|Price|Цена товара|

### Первичный ключ

- CatalogNum
    

### NOT NULL

- Product
    
- Price
    

---

# Таблица «Orders»

|Поле|Описание|
|---|---|
|OrderNum|Номер заказа|
|CustomerNum|Номер заказчика|

### Первичный ключ

- OrderNum
    

### Внешний ключ

- CustomerNum → Customer(CustomerNum)
    

### NOT NULL

- CustomerNum
    

---

# Таблица «OrderItem»

|Поле|Описание|
|---|---|
|OrderNum|Номер заказа|
|CatalogNum|Номер товара|
|Quantity|Количество|

### Первичный ключ

- (OrderNum, CatalogNum)
    

### Внешние ключи

- OrderNum → Orders(OrderNum)
    
- CatalogNum → Product(CatalogNum)
    

### NOT NULL

- Quantity
    

---

# Связи между отношениями

|Главная таблица|Подчиненная таблица|Тип связи|
|---|---|---|
|Customer|Orders|Один-ко-многим|
|Orders|OrderItem|Один-ко-многим|
|Product|OrderItem|Один-ко-многим|

---

# Соответствие 3НФ

База данных соответствует 3НФ, так как:

- отсутствуют повторяющиеся группы;
    
- все неключевые атрибуты зависят только от первичного ключа;
    
- отсутствуют транзитивные зависимости.

## Задание 4.2

### Простые запросы на SQL

---

## Запрос 1 — Вывод всех авторов

```sql
SELECT Book.Author
FROM Book;
```

---

## Запрос 2 — Авторы без повторений

```sql
SELECT DISTINCT Book.Author
FROM Book;
```

---

## Запрос 3 — Авторы без повторений в алфавитном порядке

```sql
SELECT DISTINCT Book.Author
FROM Book
ORDER BY Book.Author ASC;
```

---

## Запрос 4 — Авторы и годы издания

```sql
SELECT DISTINCT Book.Author, Book.BookYear
FROM Book
ORDER BY Book.Author ASC, Book.BookYear DESC;
```

---

## Запрос 5 — Книги от старых к новым

```sql
SELECT *
FROM Book
ORDER BY Book.BookYear ASC, Book.Title ASC;
```

---

## Запрос 6 — Две последние книги

### Для Microsoft Access

```sql
SELECT TOP 2 *
FROM Book
ORDER BY Book.BookYear DESC;
```

### Для MySQL

```sql
SELECT *
FROM Book
ORDER BY Book.BookYear DESC
LIMIT 2;
```

---

## Запрос 7 — Запрос к связанным таблицам

```sql
SELECT BookInLib.LibID, Book.Author, Book.Title
FROM Book
INNER JOIN BookInLib
ON Book.BookID = BookInLib.BookID;
```

---

## Запрос 8 — Левое соединение таблиц

```sql
SELECT BookInLib.LibID, Book.Author, Book.Title
FROM Book
LEFT JOIN BookInLib
ON Book.BookID = BookInLib.BookID;
```

---

## Запрос 9 — Книги с 1997 по 2001 год

```sql
SELECT Book.Author, Book.Title, Book.Publisher, Book.BookYear
FROM Book
WHERE Book.BookYear BETWEEN 1997 AND 2001;
```

---

## Запрос 10 — Использование IN

```sql
SELECT Book.Author, Book.Title, Book.Publisher, Book.BookYear
FROM Book
WHERE Book.BookYear BETWEEN 1997 AND 2001
AND Book.Publisher IN ('Азбука', 'Политехника');
```

---

## Запрос 11 — Использование OR

```sql
SELECT Book.Author, Book.Title, Book.Publisher, Book.BookYear
FROM Book
WHERE Book.BookYear BETWEEN 1997 AND 2001
AND (Book.Publisher = 'Азбука'
OR Book.Publisher = 'Политехника');
```

---

## Запрос 12 — Использование LIKE

### Для Microsoft Access

```sql
SELECT Book.Title
FROM Book
WHERE Book.BookYear > 1999
AND (Book.Author LIKE 'Г*'
OR Book.Publisher LIKE '*а')
ORDER BY Book.Title DESC;
```

### Для MySQL

```sql
SELECT Book.Title
FROM Book
WHERE Book.BookYear > 1999
AND (Book.Author LIKE 'Г%'
OR Book.Publisher LIKE '%а')
ORDER BY Book.Title DESC;
```

---

# Различия между Microsoft Access и MySQL

|Особенность|Microsoft Access|MySQL|
|---|---|---|
|LIKE|Используются символы `*`|Используются символы `%`|
|Ограничение количества строк|TOP|LIMIT|
|Тип COUNTER|COUNTER|AUTO_INCREMENT|
