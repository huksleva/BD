# Инвариантная часть

## Задание 2.1

### Визуализация примера для моделей и подходов к организации данных

|№|Модель / подход|Предметная область|Взаимоотношения объектов|
|---|---|---|---|
|1|Иерархическая модель|Организационная структура предприятия|Руководитель управляет отделами, отделы содержат сотрудников. Связь типа «родитель-потомок».|
|2|Сетевая модель|Система бронирования авиабилетов|Один пассажир может иметь несколько билетов, один рейс включает множество пассажиров. Связь «многие ко многим».|
|3|Реляционная модель|Университет|Таблицы «Студенты», «Группы», «Преподаватели», «Дисциплины» связаны через ключи.|
|4|Объектно-ориентированная модель|Интернет-магазин|Объекты «Товар», «Покупатель», «Заказ» имеют свойства и методы.|
|5|Документо-ориентированная модель|Социальная сеть|Информация о пользователях хранится в виде документов JSON.|
|6|Ключ-значение|Кэширование данных сайта|Каждому ключу соответствует одно значение.|
|7|Графовая модель|Социальная сеть друзей|Пользователи связаны между собой отношениями «дружба», «подписка».|

# Вариативная часть

## Задание 2.2

### Таблица «Преимущества и недостатки моделей данных»

|№|Модель данных|Преимущества|Недостатки|
|---|---|---|---|
|1|Иерархическая|Высокая скорость доступа к данным, простая структура|Сложность изменения структуры, ограниченные связи|
|2|Сетевая|Поддержка сложных связей между объектами|Сложность проектирования и сопровождения|
|3|Реляционная|Простота работы с таблицами, использование SQL, высокая надежность|Снижение производительности при больших объемах данных|
|4|Объектно-ориентированная|Удобство работы со сложными объектами|Высокая сложность реализации|
|5|Документо-ориентированная|Гибкая структура данных, хорошая масштабируемость|Возможна избыточность данных|
|6|Ключ-значение|Очень высокая скорость обработки|Ограниченные возможности сложных запросов|
|7|Графовая|Эффективная работа со связями между объектами|Используется для узкого круга задач|
|8|Многомерная|Удобство аналитики и OLAP-обработки|Требует больших вычислительных ресурсов|
