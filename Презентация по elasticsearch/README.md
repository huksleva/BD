В рамках работы был подготовлен и реализован демонстрационный проект **MovieSearch**, предназначенный для сравнения двух подходов к поиску данных:

- обычного поиска через SQL `LIKE`;
    
- полнотекстового поиска с использованием Elasticsearch.
    

### Выполненные задачи

- Реализован поиск фильмов через PostgreSQL с использованием `LIKE` и сортировки по рейтингу.
    
- Подключён и настроен Elasticsearch для полнотекстового поиска.
    
- Создан поисковый индекс `movies_index`.
    
- Настроен `russian_analyzer` для обработки русскоязычного текста:
    
    - tokenizer,
        
    - lowercase,
        
    - stemming.
        
- Реализована индексация фильмов из PostgreSQL в Elasticsearch.
    
- Настроен поиск по нескольким полям (`movie`, `overview`) с разными весами релевантности.
    
- Добавлен фразовый поиск (`match_phrase`) и relevance scoring.
    
- Подготовлено сравнение двух подходов:
    
    - качество поиска,
        
    - поддержка морфологии,
        
    - релевантность,
        
    - архитектурные отличия,
        
    - эксплуатационные затраты.
        
- Подготовлена презентация и демонстрационный сценарий для показа работы системы.
    

### Полученные результаты

В ходе работы было показано, что:

- SQL `LIKE` подходит для простых фильтров и небольших объёмов данных;
    
- Elasticsearch обеспечивает более качественный пользовательский поиск:
    
    - учитывает формы слов,
        
    - поддерживает релевантность,
        
    - ранжирует результаты,
        
    - улучшает поиск по текстовым данным.
        

### Итог

Было проведено практическое сравнение обычного поиска и поискового движка Elasticsearch на примере Spring Boot приложения. Получен опыт настройки индексации, analyzer, relevance scoring и интеграции поискового движка с PostgreSQL.
